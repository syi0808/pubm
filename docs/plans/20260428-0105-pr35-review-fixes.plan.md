## Interfaces, Data Flow, And State

- `createTaskRunner()` forwards the optional parent reference into `PubmTaskRunner`, using the same constructor path logic already used by `ContextWrapper.newTaskRunner()`.
- `createListr()` remains a compatibility wrapper around `createTaskRunner()` with the same optional parent argument already present in its signature.
- `PubmTaskRunner.add()` constructs `RuntimeTask` instances with the same path calculation as the constructor.
- Prompt capture row state remains in the renderer's existing `rows`, `row`, and `column` fields; erase modes mutate row content without introducing new state.
- Terminal wrapping continues to parse terminal controls first, then counts display columns only for visible characters.

## Edge Cases And Failure Modes

- Root task `add()` must not accidentally gain an empty parent prefix.
- `eraseLine(1)` at a column beyond the current line length must not add excess padding beyond the current cursor cell contract.
- `eraseScreen(1)` at row zero must still blank through the cursor and preserve the cursor.
- Wide characters at wrap boundaries should form lines whose display width does not exceed the requested columns unless one character itself is wider than the column count.
- Combining marks and variation selectors should not trigger a false visible-text suffix that creates empty wrapped lines.
