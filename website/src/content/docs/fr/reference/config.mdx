Cette reference decrit la forme typee de `pubm.config.*` exposee par `defineConfig()`.

Champs les plus importants aujourd'hui:

- `packages`
- `plugins`
- `versioning`, `fixed`, `linked`
- `snapshotTemplate`

Les registries sont inferees depuis les fichiers de manifeste. `packages[].registries` permet de les surcharger ou d'ajouter des registries privees.

## Defauts principaux

- `versioning: "independent"`
- `branch: "main"`
- `tag: "latest"`
- `contents: "."`
- `saveToken: true`
- `releaseDraft: true`

## Champs utiles

- `packages`
- `fixed`
- `linked`
- `plugins`
- `validate`
- `snapshotTemplate`

## `snapshotTemplate`

- Type : `string`
- Défaut : `"{tag}-{timestamp}"`

Template pour la chaîne de version lors de l'utilisation du flag `--snapshot`.

Variables disponibles :

- `{base}` — version de base actuelle depuis le fichier manifeste
- `{tag}` — tag du snapshot (par défaut : `snapshot`)
- `{timestamp}` — timestamp Unix au moment du build
- `{commit}` — hash court du commit Git
