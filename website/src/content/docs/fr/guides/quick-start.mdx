`pubm` est un orchestrateur de release pour les depots qui publient vers plusieurs registres, plusieurs paquets ou plusieurs ecosystemes.

## Avant de commencer

- Git `2.11+`
- Un manifeste comme `package.json`
- Un working tree propre avant publication

Installation :

```bash
npm i -g pubm
```

## Pour les agents de code

Si vous branchez `pubm` via Claude Code, Codex ou un autre agent, commencez par l'automatisation de setup plutot que par des prompts de release ecrits a la main.

- Utilisez le skill `publish-setup` du bundle `pubm plugin`.
- Donnez a l'agent le lien [INSTALLATION.md](/pubm/plugins/pubm-plugin/INSTALLATION.md) pour qu'il recupere le bundle et commence par `publish-setup`.
- Lisez [Integration avec les agents de code](/pubm/fr/guides/coding-agents/).

## 1. Initialiser le depot

```bash
pubm init
```

## 2. Publier

```bash
pubm
```

Si des changesets sont en attente, `pubm` les consomme pendant l'etape de version du pipeline de release. Pour la configuration, lisez [Configuration](/pubm/fr/guides/configuration/). Pour la CI, lisez [CI/CD](/pubm/fr/guides/ci-cd/).
