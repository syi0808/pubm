`pubm` est un orchestrateur de release pour les depots qui publient vers plusieurs registres, plusieurs paquets ou plusieurs ecosystemes.

## Avant de commencer

- Git `2.11+`
- Un manifeste tel que `package.json`
- Un working tree propre avant publication

Installation:

```bash
npm i -g pubm
```

## 1. Initialiser le depot

```bash
pubm init
```

## 2. Ajouter une configuration minimale

`pubm` infere les registries depuis les fichiers de manifeste. La config peut etre vide pour la plupart des projets:

```ts
import { defineConfig } from "@pubm/core";

export default defineConfig({});
```

## 3. Ajouter un changeset

```bash
pubm changesets add
```

## 4. Ecrire les versions

```bash
pubm changesets version
```

## 5. Publier

```bash
pubm
```

Le reste de la documentation utilise encore l'anglais comme fallback. Vous pouvez continuer avec [Configuration](/pubm/fr/guides/configuration/) et [CI/CD](/pubm/fr/guides/ci-cd/).
