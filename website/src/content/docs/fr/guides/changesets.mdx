---
title: Changesets
description: Enregistrer l'intention de release, generer le changelog et controler les bumps de version.
---

`pubm` utilise les fichiers dans `.pubm/changesets/` comme source durable de ce qui doit etre publie ensuite.

## Emplacement

```text
.pubm/changesets/
```

```bash
pubm init
pubm init --changesets
```

## Creer un changeset

```bash
pubm changesets add
```

## Inspecter les changements en attente

```bash
pubm changesets status
pubm changesets status --verbose
pubm changesets version --dry-run
```

## Consommation des changesets

Quand vous lancez `pubm`, les changesets en attente sont consommes pendant l'etape de version :

1. lecture des fichiers `.pubm/changesets/*.md`
2. calcul des versions finales
3. mise a jour des manifestes
4. generation du changelog
5. suppression des fichiers consommes
6. creation du commit et des tags de release

Si vous voulez n'executer que la partie versioning a l'avance, vous pouvez toujours lancer :

```bash
pubm changesets version
```
