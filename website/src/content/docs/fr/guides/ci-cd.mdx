`pubm` se comporte bien autant dans un terminal local qu'en CI. Le point cle est que le pipeline normal sait deja consommer les changesets pendant la version, donc vous n'avez pas besoin d'une etape separee `pubm changesets version` juste pour preparer la CI.

## Modele recommande

1. lancer `pubm --preflight` localement ou dans un job controle
2. publier depuis la CI avec `pubm --ci`

```bash
pubm --preflight
pubm --ci
```

## Ce que fait preflight

`pubm --preflight` :

- collecte les tokens de registre
- peut les synchroniser vers GitHub Secrets
- execute les checks prerequis et required conditions
- lance tests et build sauf si ignores
- consomme les changesets en attente pendant la version si presents
- cree le commit de release et pousse branche et tags
- dry-run les taches de publish au lieu du publish reel

Vous n'avez donc pas besoin de lancer `pubm patch --preview` apres preflight.
