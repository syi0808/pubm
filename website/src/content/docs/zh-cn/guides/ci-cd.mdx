`pubm` 同时适用于本地终端和 CI runner。关键点是常规 pipeline 已经会在 version 阶段消费待处理 changeset。

## 推荐流程

```bash
pubm --preflight
pubm --ci
```

## preflight 会做什么

- 收集 registry token
- 可选同步到 GitHub Secrets
- 执行 checks、tests 和 build
- 在 version 阶段消费待处理 changeset
- 创建 release commit，并 push branch 和 tags
- 对 publish 做 dry-run，而不是真正发布

因此 preflight 之后不需要再执行 `pubm patch --preview`。
