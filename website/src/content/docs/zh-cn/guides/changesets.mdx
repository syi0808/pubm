`pubm` 使用 `.pubm/changesets/` 中的文件作为“下一次要发布什么”的持久记录。

## 基本命令

```bash
pubm init
pubm init --changesets
pubm changesets add
pubm changesets status --verbose
pubm changesets version --dry-run
```

## 在 release 流程中消费 changeset

执行 `pubm` 时，待处理 changeset 会在 version 阶段被自动消费。`pubm changesets version` 仍然适用于受控流程，但在 `pubm` 或 `pubm --ci` 之前并不是必需步骤。
