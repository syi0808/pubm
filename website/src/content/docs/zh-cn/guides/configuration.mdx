`pubm` 是 CLI-first 工具，但真实项目很快就会超出纯 flag 配置。`pubm.config.*` 用来放仓库级 publish 目标、包布局、插件和 changeset 版本规则。

## 配置文件发现顺序

1. `pubm.config.ts`
2. `pubm.config.mts`
3. `pubm.config.cts`
4. `pubm.config.js`
5. `pubm.config.mjs`
6. `pubm.config.cjs`

TypeScript 配置文件可直接使用。

## 最小配置

```ts
import { defineConfig } from "@pubm/core";

export default defineConfig({
  registries: ["npm", "jsr"],
});
```

## config 与 CLI flag 的关系

- config 提供 `registries`、`packages`、`plugins` 等默认值
- `versioning`、`fixed`、`linked` 用于 changeset 流程
- 实际 publish 行为仍以 CLI flag 为准
- 如果都没设置，默认 registry 是 `["npm", "jsr"]`

例如：

```bash
pubm patch --registry npm --tag beta
```

## 常见仓库级选项

```ts
import { defineConfig } from "@pubm/core";
import { externalVersionSync } from "@pubm/plugin-external-version-sync";

export default defineConfig({
  registries: ["npm", "jsr"],
  plugins: [
    externalVersionSync({
      targets: [
        {
          file: "README.md",
          pattern: /pubm@([\\w.-]+)/,
        },
      ],
    }),
  ],
  snapshot: {
    useCalculatedVersion: false,
  },
});
```

## 多包配置

```ts
export default defineConfig({
  versioning: "independent",
  packages: [
    { path: "packages/core", registries: ["npm", "jsr"], ecosystem: "js" },
    { path: "packages/cli", registries: ["npm"], ecosystem: "js" },
    { path: "crates/pubm-rs", registries: ["crates"], ecosystem: "rust" },
  ],
});
```

## 版本策略

### `independent`

每个包单独拥有版本和 tag。

### `fixed`

所有受影响包共享同一版本。

### `fixed` / `linked`

```ts
export default defineConfig({
  versioning: "independent",
  fixed: [["@acme/core", "@acme/react"]],
  linked: [["@acme/cli", "@acme/config"]],
});
```

## snapshot

```ts
export default defineConfig({
  snapshot: {
    useCalculatedVersion: true,
    prereleaseTemplate: "{tag}-{timestamp}",
  },
});
```

## 插件

插件通过 `plugins` 数组注册。关于 hook 顺序和自定义 registry，请阅读 [Plugins API Reference](/pubm/zh-cn/reference/plugins/)。
