`pubm` 是面向多 registry、多包、多生态发布场景的 release orchestration 工具。

## 开始前

- Git `2.11+`
- `package.json` 之类的包清单
- 发布前保持干净的 working tree

安装 CLI：

```bash
npm i -g pubm
```

## 面向编码代理

如果你通过 Claude Code、Codex 或其他编码代理接入 `pubm`，优先从 setup 自动化开始，而不是手写 release prompt。

- 使用 `publish-setup` skill。
- 把 [INSTALLATION.md](/pubm/plugins/pubm-plugin/INSTALLATION.md) 链接提供给代理，让它获取 bundle 并从 `publish-setup` 开始。
- 阅读 [Coding Agent Integration](/pubm/zh-cn/guides/coding-agents/)。

## 1. 初始化仓库

```bash
pubm init
```

## 2. 发布

```bash
pubm
```

如果存在待处理 changeset，`pubm` 会在 release pipeline 的 version 阶段自动消费它们。
