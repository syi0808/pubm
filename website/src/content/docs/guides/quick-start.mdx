`pubm` is a release orchestrator for projects that publish to more than one registry, more than one package, or more than one ecosystem.

## Before you start

`pubm` is designed for repositories that already use Git and have a valid package manifest.

- Git `2.11+`
- A package manifest such as `package.json`
- A clean working tree before you publish

`pubm` itself is distributed as a Bun-built single binary. You do not need a separate Node.js runtime just to run the CLI, although npm-based installation still uses your package manager toolchain.

Install the CLI globally:

```bash
npm i -g pubm
```

If you are using Claude Code or another coding agent, read [Coding Agent Integration](/pubm/guides/coding-agents/) before wiring release automation.

## 1. Initialize the repository

Create the basic `pubm` layout:

```bash
pubm init
```

This creates `pubm.config.ts` and `.pubm/changesets/`.

## 2. Add a minimal config

Set the registries you plan to publish to:

```ts
import { defineConfig } from "@pubm/core";

export default defineConfig({
  registries: ["npm", "jsr"],
});
```

For config details, read [Configuration](/pubm/guides/configuration/).

## 3. Add a changeset

Create a changeset:

```bash
pubm changesets add
```

## 4. Write versions

Apply pending changesets:

```bash
pubm changesets version
```

You can inspect pending changes and the resulting versions with:

```bash
pubm changesets status --verbose
pubm changesets version --dry-run
```

If you want to validate registry auth and dry-run publish steps first, run:

```bash
pubm --preflight
```

## 5. Publish

Start the release flow:

```bash
pubm
```

Or choose an explicit version bump:

```bash
pubm patch
```

For CI, tokens, and preflight behavior, read [CI/CD](/pubm/guides/ci-cd/). For monorepos, read [Monorepo](/pubm/guides/monorepo/). For exact flags and subcommands, read the [CLI Reference](/pubm/reference/cli/).
