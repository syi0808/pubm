`pubm` is a release orchestrator for projects that publish to more than one registry, more than one package, or more than one ecosystem.

## Before you start

`pubm` is designed for repositories that already use Git and have a valid package manifest.

- Git `2.11+`
- A package manifest such as `package.json`
- A clean working tree before you publish

`pubm` itself is distributed as a Bun-built single binary. You do not need a separate Node.js runtime just to run the CLI, although npm-based installation still uses your package manager toolchain.

Install the CLI globally:

```bash
npm i -g pubm
```

## For coding agents

If you are wiring `pubm` through Claude Code, Codex, or another coding agent, start with the setup automation instead of hand-writing release prompts.

- Use the `publish-setup` skill from the `pubm plugin` bundle to install the release workflow, generate config, wire CI, and enable changesets.
- Give your agent the [INSTALLATION.md](/pubm/plugins/pubm-plugin/INSTALLATION.md) link so it can fetch the bundle and start with `publish-setup`.
- Read [Coding Agent Integration](/pubm/guides/coding-agents/) for the integration model and skill roles.

## 1. Initialize the repository

Create the basic `pubm` layout:

```bash
pubm init
```

This creates `pubm.config.ts` and `.pubm/changesets/`.

## 2. Publish

Start the release flow:

```bash
pubm
```

Or choose an explicit version bump:

```bash
pubm patch
```

If pending changesets exist, `pubm` consumes them during the version step of the release pipeline. For config details, read [Configuration](/pubm/guides/configuration/). For CI, tokens, and CI preparation behavior, read [CI/CD](/pubm/guides/ci-cd/). For monorepos, read [Monorepo](/pubm/guides/monorepo/). For exact flags and subcommands, read the [CLI Reference](/pubm/reference/cli/).
