title: CI/CD
description: Configurar preflight, manejo de tokens y automatizacion de releases para pubm.
---

`pubm` funciona bien tanto en terminal local como en CI. El pipeline normal ya puede consumir los changesets pendientes durante la version.

## Flujo recomendado

```bash
pubm --preflight
pubm --ci
```

## Que hace preflight

- recopila tokens de registry
- puede sincronizarlos a GitHub Secrets
- ejecuta checks, tests y build
- consume changesets pendientes durante versionado
- crea el commit de release y hace push de rama y tags
- hace dry-run del publish real

Por eso no hace falta ejecutar `pubm patch --preview` despues de preflight.
