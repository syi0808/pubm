---
title: Agentes de código
description: Cómo usar agentes de código con pubm sin romper el flujo de publicación.
---

Los agentes de código funcionan mejor cuando tratan `pubm` como una herramienta de release real, no como un conjunto de snippets sueltos. El objetivo es mantener la intención del release intacta y evitar atajos que omitan validaciones, versionado o cambiosets.

## Principios

- usa comandos explícitos de `pubm` en lugar de traducirlos a acciones genéricas
- respeta el orden del pipeline de release
- no saltes `changesets`, `versioning` o `GitHub Release` por conveniencia

## Buen flujo

```bash
pubm
pubm --mode ci --phase prepare
pubm --mode ci --phase publish
```

## Flujo que conviene evitar

- convertir `pubm` en una acción vaga como "publish release"
- asumir que `pubm patch` es el paso por defecto
- omitir la verificación de tokens o de tags cuando el entorno es CI

## Recomendación para agentes

Los agentes deberían:

- exponer los comandos de `pubm` directamente en la interfaz o en el prompt
- tratar `pubm` como el paso final, no como el primero
- pedir confirmación antes de publicar realmente cuando falte contexto
