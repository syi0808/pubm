`pubm` es un orquestador de releases para repositorios que publican en varios registros, varios paquetes o varios ecosistemas.

## Antes de empezar

- Git `2.11+`
- Un manifiesto como `package.json`
- Un working tree limpio antes de publicar

Instalacion:

```bash
npm i -g pubm
```

## Para agentes de codigo

Si conectas `pubm` mediante Claude Code, Codex u otro agente, empieza por la automatizacion de setup.

- Usa el skill `publish-setup`.
- Dale al agente el enlace [INSTALLATION.md](/pubm/plugins/pubm-plugin/INSTALLATION.md) para que obtenga el bundle y empiece con `publish-setup`.
- Lee [Integracion con agentes de codigo](/pubm/es/guides/coding-agents/).

## 1. Inicializar el repositorio

```bash
pubm init
```

## 2. Publicar

```bash
pubm
```

Si hay changesets pendientes, `pubm` los consume durante la etapa de version del pipeline.
