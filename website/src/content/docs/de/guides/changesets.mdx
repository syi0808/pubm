---
title: Changesets
description: Release-Vorhaben festhalten, Changelogs erzeugen und Versionsspruenge steuern.
---

`pubm` nutzt Changesets als dauerhafte Aufzeichnung dafuer, was als Nächstes veröffentlicht werden soll. Statt Versionen aus Commit-Meldungen oder Branch-Namen abzuleiten, liest es versionierte Dateien aus `.pubm/changesets/` und wandelt sie in Versionsspruenge und Changelog-Eintraege um.

## Why changesets matter

- ein pruefbarer Release-Plan in jedem Pull Request
- deterministische Berechnung von Versionsspruengen ueber mehrere Pakete hinweg
- Changelog-Inhalte, die vor dem Release-Tag geschrieben werden, nicht erst danach

## File location

Pending changesets live in:

```text
.pubm/changesets/
```

Initialisiere dieses Verzeichnis, indem du den interaktiven Setup-Wizard ausfuehrst:

```bash
pubm init
```

Waehrend des Setups fragt `pubm init`, ob der Changesets-Workflow aktiviert werden soll. Bei Ja erstellt es das Verzeichnis `.pubm/changesets/`, erzeugt `.github/workflows/changeset-check.yml` fuer die PR-Durchsetzung und aktualisiert `.gitignore`, damit Changeset-Dateien nachverfolgt, andere `.pubm/`-Inhalte aber ignoriert werden.

## Creating a changeset

```bash
pubm changesets add
```

In a monorepo, `pubm` also asks which packages are affected and what bump type each one should receive.

For non-interactive use:

```bash
pubm changesets add \
  --packages @acme/core,@acme/react \
  --bump minor \
  --message "Add token-based auth to the public client."
```

## What a changeset file contains

```md
---
"@acme/core": minor
"@acme/react": patch
---

Add token-based auth to the public client.
```

The metadata controls version calculation. The body becomes the changelog summary.

## Inspecting pending changes

Before consuming changesets, inspect them:

```bash
pubm changesets status
pubm changesets status --verbose
pubm changesets version --dry-run
```

## How changesets are consumed

Wenn du die normale Release-Pipeline mit `pubm` startest, werden ausstehende Changesets waehrend des Versionsschritts verbraucht.

That flow:

1. liest ausstehende `.pubm/changesets/*.md`-Dateien
2. berechnet die resultierenden Paketversionen
3. aktualisiert Manifest-Versionen
4. erzeugt Changelog-Eintraege
5. loescht verbrauchte Changeset-Dateien
6. erstellt den Release-Commit und die Tags

If you want to perform only the versioning part ahead of time, you can still run:

```bash
pubm changesets version
```

Der Befehl ist fuer redaktionelle oder kontrollierte Workflows nuetzlich, aber nicht erforderlich vor `pubm` oder `pubm --mode ci --phase publish`.

## Changelog generation

If you want to generate changelog text without consuming versions yet:

```bash
pubm changesets changelog --dry-run
pubm changesets changelog --version 1.8.0
```

## Migration from `.changeset/`

```bash
pubm changesets migrate
```

This migrates pending files from `.changeset/` into `.pubm/`.
