`pubm` ist CLI-first, aber echte Projekte brauchen schnell repository-weite Konfiguration uber `pubm.config.*`.

## Minimales Beispiel

`pubm` leitet Registries aus den Manifest-Dateien ab. Fur die meisten Projekte reicht eine leere Konfiguration:

```ts
import { defineConfig } from "@pubm/core";

export default defineConfig({});
```

## Registry-Ableitung

| Manifest | Abgeleitete Registry |
|---|---|
| `package.json` | `npm` |
| `jsr.json` | `jsr` |
| `Cargo.toml` | `crates` |

## Wichtige Felder

- `packages`
- `plugins`
- `versioning`
- `fixed`
- `linked`
- `snapshot`
