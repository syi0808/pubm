`pubm` funktioniert in lokalen Terminals und in CI. Der normale Pipeline-Flow kann ausstehende Changesets bereits wahrend der Version konsumieren.

## Empfohlenes Modell

```bash
pubm --preflight
pubm --ci
```

## Was preflight tut

- sammelt Registry-Tokens
- kann sie nach GitHub Secrets synchronisieren
- fuhrt Prufungen, Tests und Build aus
- konsumiert Changesets im Version-Schritt, wenn welche vorliegen
- erstellt den Release-Commit und pusht Branch und Tags
- fuhrt statt echtem Publish einen Dry-Run aus

Darum ist `pubm patch --preview` nach preflight nicht mehr notig.
