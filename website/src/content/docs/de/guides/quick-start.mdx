`pubm` ist ein Release-Orchestrator fur Repositories, die in mehrere Registries, mehrere Pakete oder mehrere Okosysteme veroffentlichen.

## Vor dem Start

- Git `2.11+`
- Ein Manifest wie `package.json`
- Ein sauberer Working Tree vor dem Publish

Installation:

```bash
npm i -g pubm
```

## Fur Coding Agents

Wenn du `pubm` uber Claude Code, Codex oder einen anderen Agent anschliesst, starte mit der Setup-Automatisierung statt mit handgeschriebenen Release-Prompts.

- Nutze den Skill `publish-setup` aus dem `pubm plugin`-Bundle.
- Gib dem Agent den Link [INSTALLATION.md](/pubm/plugins/pubm-plugin/INSTALLATION.md), damit er das Bundle holen und mit `publish-setup` starten kann.
- Lies [Integration mit Coding Agents](/pubm/de/guides/coding-agents/).

## 1. Repository initialisieren

```bash
pubm init
```

## 2. Publish ausfuhren

```bash
pubm
```

Wenn Changesets ausstehen, konsumiert `pubm` sie im Version-Schritt der Release-Pipeline.
