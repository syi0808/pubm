`pubm` ist ein Release-Orchestrator fur Repositories, die in mehrere Registries, mehrere Pakete oder mehrere Okosysteme veroffentlichen.

## Vor dem Start

- Git `2.11+`
- Ein Manifest wie `package.json`
- Ein sauberer Working Tree vor dem Publish

Installation:

```bash
npm i -g pubm
```

## 1. Repository initialisieren

```bash
pubm init
```

## 2. Minimale Konfiguration anlegen

`pubm` leitet Registries aus den Manifest-Dateien ab. Fur die meisten Projekte reicht eine leere Konfiguration:

```ts
import { defineConfig } from "@pubm/core";

export default defineConfig({});
```

## 3. Changeset hinzufugen

```bash
pubm changesets add
```

## 4. Versionen schreiben

```bash
pubm changesets version
```

## 5. Publish ausfuhren

```bash
pubm
```
