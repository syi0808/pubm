`pubm` ist ein Release-Orchestrator fuer Projekte, die in mehr als eine Registry, mehr als ein Paket oder mehr als ein Ecosystem veroeffentlichen.

## Before you start

`pubm` ist fuer Repositories gedacht, die bereits Git nutzen und ein gueltiges Paket-Manifest haben.

- Git `2.11+`
- ein Paket-Manifest wie `package.json`
- ein sauberer Working Tree vor dem Veröffentlichen

`pubm` selbst wird als von Bun gebautes Einzelbinary ausgeliefert. Du brauchst keine separate Node.js-Laufzeit nur fuer die CLI, auch wenn die npm-basierte Installation weiterhin deine Paketmanager-Toolchain verwendet.

Installiere die CLI global:

```bash
npm i -g pubm
```

## For coding agents

Wenn du `pubm` ueber Claude Code, Codex oder einen anderen Coding Agent einbindest, beginne mit der Setup-Automatisierung statt Release-Prompts von Hand zu schreiben.

- Fuehre `pubm setup-skills` aus, um Coding-Agent-Skills direkt herunterzuladen und zu installieren, oder lass `pubm init` die Skill-Installation als letzten Schritt anbieten.
- Alternativ verwende den Skill `publish-setup` aus dem `pubm plugin`-Bundle, um den Release-Workflow zu installieren, Konfiguration zu erzeugen, CI anzubinden und Changesets zu aktivieren.
- Gib deinem Agenten den Link [INSTALLATION.md](/pubm/de/plugins/pubm-plugin/INSTALLATION.md), damit er das Bundle abrufen und mit `publish-setup` starten kann.
- Lies [Coding Agent Integration](/pubm/de/de/guides/coding-agents/) fuer das Integrationsmodell und die Rollen der Skills.

## 1. Initialize the repository

Starte den interaktiven Setup-Wizard:

```bash
pubm init
```

Der Wizard erkennt deine Pakete und Registries und fuehrt dich dann durch Branch-Konfiguration, Versionierungsstrategie, Changelog, Changesets, CI-Workflow-Erzeugung und Coding-Agent-Skills. Eine Config-Datei wird nur erstellt, wenn deine Auswahl von den Standardwerten abweicht.

## 2. Publish

Starte den Release-Flow:

```bash
pubm
```

Wenn ausstehende Changesets existieren, verbraucht `pubm` sie waehrend des Versionsschritts der Release-Pipeline. Fuer Konfigurationsdetails lies [Configuration](/pubm/de/de/guides/configuration/). Fuer CI, Tokens und das Verhalten der CI-Vorbereitung lies [CI/CD](/pubm/de/de/guides/ci-cd/). Fuer Monorepos lies [Monorepo](/pubm/de/de/guides/monorepo/). Fuer exakte Flags und Subcommands lies die [CLI Reference](/pubm/de/de/reference/cli/).
