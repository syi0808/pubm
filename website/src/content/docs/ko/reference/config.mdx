title: Config API Reference
description: "`pubm.config.*`에서 지원하는 필드와 기본값, 사용 메모를 설명합니다."
---

이 문서는 `@pubm/core`의 `defineConfig()`가 노출하는 `pubm.config.*` 타입 구조를 설명합니다.

현재 실제 publish 흐름에서 특히 중요한 필드는 다음입니다.

- `registries`
- `packages`
- `plugins`
- `versioning`, `fixed`, `linked`
- `snapshot.useCalculatedVersion`

## 기본 설정

```ts
export default defineConfig({
  versioning: "independent",
  branch: "main",
  changelog: true,
  changelogFormat: "default",
  commit: false,
  access: "public",
  fixed: [],
  linked: [],
  updateInternalDependencies: "patch",
  ignore: [],
  tag: "latest",
  contents: ".",
  saveToken: true,
  releaseDraft: true,
  releaseNotes: true,
  registries: ["npm", "jsr"],
  rollbackStrategy: "individual",
  packages: [{ path: ".", registries: ["npm", "jsr"] }],
  validate: {
    cleanInstall: true,
    entryPoints: true,
    extraneousFiles: true,
  },
  snapshot: {
    useCalculatedVersion: false,
    prereleaseTemplate: "{tag}-{timestamp}",
  },
});
```

## 주요 top-level 옵션

### `versioning`

- 타입: `"independent" | "fixed"`
- 기본값: `"independent"`

멀티 패키지 저장소에서 버전 부여 방식을 제어합니다.

### `branch`

- 타입: `string`
- 기본값: `"main"`

가드된 릴리스에서 `HEAD`가 일치해야 하는 브랜치입니다.

### `registries`

- 타입: `RegistryType[]`
- 기본값: `["npm", "jsr"]`

기본 내장값:

- `"npm"`
- `"jsr"`
- `"crates"`
- 커스텀 레지스트리 URL

### `packages`

- 타입: `PackageConfig[]`
- 기본값: `[{ path: ".", registries: ["npm", "jsr"] }]`

```ts
packages: [
  { path: "packages/core", registries: ["npm", "jsr"], ecosystem: "js" },
  { path: "crates/pubm-rs", registries: ["crates"], ecosystem: "rust" },
];
```

### `changelog`

- 타입: `boolean | string`
- 기본값: `true`

### `access`

- 타입: `"public" | "restricted"`
- 기본값: `"public"`

npm 스타일 publish에서 접근 수준을 나타냅니다.

### `fixed`

- 타입: `string[][]`
- 기본값: `[]`

항상 같은 버전을 가져야 하는 패키지 그룹입니다.

### `linked`

- 타입: `string[][]`
- 기본값: `[]`

최고 bump 타입을 공유해야 하는 그룹입니다.

### `tag`

- 타입: `string`
- 기본값: `"latest"`

기본 dist-tag입니다.

### `contents`

- 타입: `string`
- 기본값: `"."`

publish 파이프라인 전에 `chdir`할 디렉터리입니다.

### `saveToken`

- 타입: `boolean`
- 기본값: `true`

로컬 머신에서 토큰을 재사용을 위해 저장할지 결정합니다.

### `releaseDraft`

- 타입: `boolean`
- 기본값: `true`

일반 publish 흐름에서 GitHub release draft 단계를 열지 여부입니다.

### `rollbackStrategy`

- 타입: `"individual" | "all"`
- 기본값: `"individual"`

### `plugins`

- 타입: `PubmPlugin[]`
- 기본값: `[]`

훅, 명령, 레지스트리, 생태계를 확장합니다.

### `validate`

```ts
{
  cleanInstall: true,
  entryPoints: true,
  extraneousFiles: true,
}
```

공개 스키마에는 포함되지만 현재 파이프라인에서 모두 활성 토글은 아닙니다.

### `snapshot`

```ts
{
  useCalculatedVersion: false,
  prereleaseTemplate: "{tag}-{timestamp}",
}
```
