플러그인 시스템을 사용하면 핵심 publish 파이프라인을 포크하지 않고 `pubm`을 확장할 수 있습니다.

- 주요 단계 전후에 코드 실행
- 커스텀 CLI 명령 추가
- 커스텀 레지스트리 등록
- 커스텀 생태계 등록

## 기본 형태

```ts
import type { PubmPlugin } from "@pubm/core";

const myPlugin: PubmPlugin = {
  name: "my-plugin",
  hooks: {},
  commands: [],
  registries: [],
  ecosystems: [],
};
```

## `PubmPlugin`

```ts
export interface PubmPlugin {
  name: string;
  registries?: Registry[];
  ecosystems?: Ecosystem[];
  hooks?: PluginHooks;
  commands?: PluginCommand[];
}
```

## 필수 필드

### `name`

로그, 설정, 지원 이슈에서 사용자가 참조하는 기본 식별자입니다. 안정적으로 유지하세요.

## Hooks

```ts
export interface PluginHooks {
  beforeTest?: HookFn;
  afterTest?: HookFn;
  beforeBuild?: HookFn;
  afterBuild?: HookFn;
  beforeVersion?: HookFn;
  afterVersion?: HookFn;
  beforePublish?: HookFn;
  afterPublish?: HookFn;
  beforePush?: HookFn;
  afterPush?: HookFn;
  afterRelease?: AfterReleaseHookFn;
  onError?: ErrorHookFn;
  onRollback?: HookFn;
  onSuccess?: HookFn;
}
```

일반 로컬 publish 흐름에서는 `beforeTest`에서 시작해 `onSuccess`까지 순서대로 실행됩니다. 실패하면 `onError`, rollback, `onRollback`이 이어집니다.

## Hook 시그니처

```ts
type HookFn = (ctx: Ctx) => Promise<void> | void;
type ErrorHookFn = (ctx: Ctx, error: Error) => Promise<void> | void;
type AfterReleaseHookFn = (
  ctx: Ctx,
  releaseCtx: ReleaseContext,
) => Promise<void> | void;
```

## `Ctx`

유용한 필드:

- `version`
- `versions`
- `registries`
- `packages`
- `tag`
- `preview`
- `preflight`
- `promptEnabled`
- `releaseContext`

## 예시: publish 후 알림

```ts
const notifyPlugin = {
  name: "notify-plugin",
  hooks: {
    async afterPublish(ctx) {
      const label =
        ctx.versions && ctx.versions.size > 0
          ? [...ctx.versions].map(([name, version]) => `${name}@${version}`).join(", ")
          : `v${ctx.version}`;

      console.log(`Published ${label}`);
    },
  },
};
```

## 예시: 브랜치 정책

```ts
const branchPolicyPlugin = {
  name: "branch-policy",
  hooks: {
    beforeVersion(ctx) {
      if (ctx.tag === "latest" && ctx.branch !== "main") {
        throw new Error("Stable releases are only allowed from main.");
      }
    },
  },
};
```

## 커스텀 명령

```ts
const myPlugin = {
  name: "my-plugin",
  commands: [
    {
      name: "notify",
      description: "수동 알림 도우미",
      subcommands: [
        {
          name: "slack",
          description: "Slack 알림 전송",
          options: [{ name: "--channel <channel>", description: "대상 채널" }],
          action: async (args) => {
            console.log(`Notify ${args.channel}`);
          },
        },
      ],
    },
  ],
};
```

```bash
pubm notify slack --channel "#releases"
```

## 커스텀 레지스트리

```ts
class InternalRegistry extends Registry {
  async ping() {}
  async publish() {}
}
```

```ts
const plugin = {
  name: "internal-registry",
  registries: [new InternalRegistry()],
};
```

## 커스텀 생태계

추가 패키지 타입의 버전 읽기/쓰기 규칙을 `pubm`에 가르치고 싶을 때 사용합니다.
