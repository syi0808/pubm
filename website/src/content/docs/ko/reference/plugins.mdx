플러그인 시스템을 사용하면 핵심 publish 파이프라인을 포크하지 않고 `pubm`을 확장할 수 있습니다.

## `PubmPlugin`

```ts
export interface PubmPlugin {
  name: string;
  registries?: Registry[];
  ecosystems?: Ecosystem[];
  hooks?: PluginHooks;
  commands?: PluginCommand[];
}
```

## Hooks

```ts
export interface PluginHooks {
  beforeTest?: HookFn;
  afterTest?: HookFn;
  beforeBuild?: HookFn;
  afterBuild?: HookFn;
  beforeVersion?: HookFn;
  afterVersion?: HookFn;
  beforePublish?: HookFn;
  afterPublish?: HookFn;
  beforePush?: HookFn;
  afterPush?: HookFn;
  afterRelease?: AfterReleaseHookFn;
  onError?: ErrorHookFn;
  onRollback?: HookFn;
  onSuccess?: HookFn;
}
```

## Hook 시그니처

```ts
type HookFn = (ctx: Ctx) => Promise<void> | void;
type ErrorHookFn = (ctx: Ctx, error: Error) => Promise<void> | void;
type AfterReleaseHookFn = (
  ctx: Ctx,
  releaseCtx: ReleaseContext,
) => Promise<void> | void;
```

## `Ctx`

훅은 현재 파이프라인 컨텍스트를 받습니다. 런타임 타입은 해석된 CLI 옵션에 내부 상태 몇 가지가 추가된 형태입니다.

```ts
export interface Ctx extends ResolvedOptions {
  promptEnabled: boolean;
  cleanWorkingTree: boolean;
  pluginRunner: PluginRunner;
  releaseContext?: ReleaseContext;
}
```

실제로 자주 보게 되는 필드:

- `version`
- `versions`
- `packages`
- `tag`
- `preview`
- `preflight`
- `ci`
- `changesetConsumed`

`Ctx`는 현재 실행의 런타임 상태로 취급하고, 영속 저장소처럼 사용하지 않는 편이 좋습니다.

## 예시

```ts
const notifyPlugin = {
  name: "notify-plugin",
  hooks: {
    async afterPublish(ctx) {
      const label =
        ctx.versions && ctx.versions.size > 0
          ? [...ctx.versions].map(([name, version]) => `${name}@${version}`).join(", ")
          : `v${ctx.version}`;

      console.log(`Published ${label}`);
    },
  },
};
```

## 커스텀 명령

```ts
export interface PluginCommand {
  name: string;
  description: string;
  subcommands?: PluginSubcommand[];
}

export interface PluginSubcommand {
  name: string;
  description: string;
  options?: PluginCommandOption[];
  action: (args: Record<string, unknown>) => Promise<void>;
}
```

## 커스텀 레지스트리와 생태계

커스텀 레지스트리는 추가 publish 대상이 필요할 때 사용합니다. 커스텀 생태계는 새로운 패키지 타입의 버전 읽기/쓰기를 `pubm`에 가르칠 때 사용합니다.
