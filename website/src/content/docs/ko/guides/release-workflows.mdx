실제 릴리스가 로컬 검사에서 publish된 패키지까지 어떻게 이동할지 정할 때 이 가이드를 사용합니다.

`pubm`은 현재 두 가지 릴리스 워크플로를 지원합니다.

| 워크플로 | 명령 | 사용할 때 |
|----------|------|----------|
| Direct Release | `pubm` | 신뢰할 수 있는 로컬 터미널이나 통제된 job 하나가 전체 릴리스를 실행해야 할 때 |
| Split CI Release | 로컬 `pubm --phase prepare`, 이후 CI `pubm --phase publish` | 로컬 준비 단계가 version과 tag를 기록하고 CI가 토큰으로 publish해야 할 때 |

## Direct Release

Direct Release는 전체 릴리스 파이프라인을 한 명령으로 실행합니다.

```bash
pubm
```

릴리스를 시작하는 사람이나 job이 registry credentials를 안전하게 보유하고, release commit과 tags를 push하며, 패키지를 publish할 수 있을 때 사용합니다.

Direct Release는 다음을 수행합니다.

- 프로젝트와 Git 상태를 검증합니다.
- 사전 조건과 필수 조건 검사를 실행합니다.
- 생략하지 않았다면 테스트와 빌드를 실행합니다.
- 보류 중인 changeset이 있으면 소비합니다.
- manifest versions를 씁니다.
- release commit과 tag를 만듭니다.
- 선택한 registries에 패키지를 publish합니다.
- 활성화된 경우 GitHub Releases를 만듭니다.
- 가능한 경우 publish-side 실패를 rollback합니다.

## Split CI Release

Split CI Release는 변경 가능한 준비 단계와 non-interactive publish 단계를 분리합니다.

```bash
pubm --phase prepare
```

```bash
pubm --phase publish
```

Maintainer가 로컬에서 릴리스를 승인하거나 준비하되, registry publish는 repository secrets가 있는 CI에서 실행해야 할 때 사용합니다.

### Prepare for CI publish

로컬에서 Prepare for CI publish를 실행합니다.

```bash
pubm --phase prepare
```

Prepare for CI publish는 다음을 수행합니다.

- registry tokens와 plugin credentials를 수집합니다.
- 필요하면 credentials를 GitHub Secrets로 동기화합니다.
- 사전 조건과 필수 조건 검사를 실행합니다.
- 생략하지 않았다면 테스트와 빌드를 실행합니다.
- 보류 중인 changeset이 있으면 versioning 중 소비합니다.
- manifest versions를 씁니다.
- release commit과 tags를 만듭니다.
- CI가 가져갈 release commit과 tag를 push합니다.
- 실제 publish 대신 publish tasks를 dry-run합니다.
- 패키지는 publish하지 않습니다.

### Publish prepared release

CI에서 Publish prepared release를 실행합니다.

```bash
pubm --phase publish
```

Publish prepared release는 다음을 수행합니다.

- package manifests에 이미 기록된 versions를 읽습니다.
- 선택한 registries에 패키지를 publish합니다.
- 설정된 경우 GitHub Releases를 만들고 release assets를 업로드합니다.
- registry와 plugin tokens를 CI 환경에서 기대합니다.
- interactive prompts 없이 실행됩니다.
- versioning을 다시 실행하지 않습니다.
- tags를 다시 만들지 않습니다.
- release commit이나 tags를 다시 push하지 않습니다.

## 현재 범위

CI가 로컬 `pubm --phase prepare` 없이 릴리스를 결정하고 준비하는 완전 자동 CI는 생성된 GitHub Actions 워크플로로 지원됩니다.

PR approval과 release-PR 흐름은 생성된 GitHub Actions 워크플로로 사용할 수 있습니다. `pubm workflow install github`로 release PR과 publish 워크플로를 설치하세요.
