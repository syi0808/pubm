`pubm`은 CLI 우선 도구이지만, 실제 프로젝트는 곧 플래그만으로는 부족해집니다. `pubm.config.*`에는 저장소 수준의 publish 대상, 패키지 구조, 플러그인, changeset 버전 규칙을 넣습니다.

## 설정 파일 탐색 순서

1. `pubm.config.ts`
2. `pubm.config.mts`
3. `pubm.config.cts`
4. `pubm.config.js`
5. `pubm.config.mjs`
6. `pubm.config.cjs`

TypeScript 설정 파일도 직접 지원합니다.

## 최소 설정

`pubm`은 매니페스트 파일에서 레지스트리를 자동 추론합니다. 대부분의 프로젝트에서는 설정을 비워둘 수 있습니다:

```ts
import { defineConfig } from "@pubm/core";

export default defineConfig({});
```

## 레지스트리 추론 방식

매니페스트 파일과 추론 결과:

| 매니페스트 | 추론되는 레지스트리 |
|---|---|
| `package.json` | `npm` |
| `jsr.json` | `jsr` |
| `Cargo.toml` | `crates` |

npm 추가 규칙:
- `package.json`의 `publishConfig.registry` → npm 대신 해당 URL 사용
- `.npmrc` 레지스트리 → `publishConfig`가 없을 때 npm 대체

## config와 CLI 플래그의 관계

- config는 `packages`, `plugins` 기본값을 제공합니다.
- `versioning`, `fixed`, `linked`는 changeset 흐름에 쓰입니다.
- 레지스트리는 매니페스트에서 추론됩니다. `packages[].registries`로 재정의 가능합니다.

예:

```bash
pubm patch --registry npm --tag beta
```

## 자주 쓰는 저장소 옵션

```ts
import { defineConfig } from "@pubm/core";
import { externalVersionSync } from "@pubm/plugin-external-version-sync";

export default defineConfig({
  plugins: [
    externalVersionSync({
      targets: [
        {
          file: "README.md",
          pattern: /pubm@([\\w.-]+)/,
        },
      ],
    }),
  ],
  snapshot: {
    useCalculatedVersion: false,
  },
});
```

## 멀티 패키지 설정

레지스트리는 각 패키지의 매니페스트 파일에서 자동 추론됩니다:

```ts
export default defineConfig({
  versioning: "independent",
  packages: [
    { path: "packages/core", ecosystem: "js" },
    { path: "packages/cli", ecosystem: "js", registries: ["npm"] }, // 재정의
    { path: "crates/pubm-rs", ecosystem: "rust" },
  ],
});
```

다음 경우 명시적 `packages` 설정이 좋습니다.

- 자동 탐지된 워크스페이스 중 일부만 배포하고 싶을 때
- JS와 Rust가 함께 있을 때
- 추론된 레지스트리를 재정의하거나 프라이빗 레지스트리를 추가할 때

## 버전 전략

### `independent`

각 패키지가 독립 버전과 태그를 가집니다.

### `fixed`

모든 영향 받은 패키지가 하나의 버전을 공유합니다.

### `fixed` / `linked` 그룹

```ts
export default defineConfig({
  versioning: "independent",
  fixed: [["@acme/core", "@acme/react"]],
  linked: [["@acme/cli", "@acme/config"]],
});
```

- `fixed`: 항상 정확히 같은 버전
- `linked`: 그룹 내 최고 bump 타입 전파

## snapshot 설정

```ts
export default defineConfig({
  snapshot: {
    useCalculatedVersion: true,
    prereleaseTemplate: "{tag}-{timestamp}",
  },
});
```

## 플러그인

```ts
import { defineConfig } from "@pubm/core";
import { externalVersionSync } from "@pubm/plugin-external-version-sync";

export default defineConfig({
  plugins: [
    externalVersionSync({
      targets: [
        {
          file: "README.md",
          pattern: /pubm@([\\w.-]+)/,
        },
      ],
    }),
  ],
});
```
