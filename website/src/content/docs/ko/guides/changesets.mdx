title: Changesets
description: 릴리스 의도를 기록하고, changelog를 만들고, 버전 상승을 제어합니다.
---

`pubm`은 다음에 무엇을 릴리스해야 하는지 `.pubm/changesets/`의 파일로 기록합니다. 커밋 메시지나 브랜치 이름을 추론하지 않고, 체크인된 changeset 파일을 기준으로 버전과 changelog를 계산합니다.

## 왜 중요한가

- PR 안에서 검토 가능한 릴리스 계획을 남길 수 있습니다.
- 패키지별 버전 계산이 결정적입니다.
- changelog 문구를 릴리스 당일이 아니라 미리 작성할 수 있습니다.

## 위치와 초기화

```text
.pubm/changesets/
```

```bash
pubm init
pubm init --changesets
```

## changeset 만들기

대화형:

```bash
pubm changesets add
```

비대화형:

```bash
pubm changesets add \
  --packages @acme/core,@acme/react \
  --bump minor \
  --message "공개 클라이언트에 토큰 인증 추가"
```

빈 파일이 필요하면:

```bash
pubm changesets add --empty
```

## 파일 형식

```md
---
"@acme/core": minor
"@acme/react": patch
---

공개 클라이언트에 토큰 인증 추가
```

frontmatter는 버전 계산에 쓰이고, 본문은 changelog 요약이 됩니다.

## bump 타입 선택

### `patch`

- 버그 수정
- 사용자 영향이 있는 문서/작은 수정
- 공개 API를 바꾸지 않는 내부 개선

### `minor`

- 하위 호환되는 기능 추가
- 새로운 CLI 플래그
- 기존 사용자를 깨지 않는 API 추가

### `major`

- 옵션/명령 제거
- 비호환 런타임 요구사항
- 사용자 코드나 설정 변경을 강제하는 동작 변경

## 대기 중 changeset 확인

```bash
pubm changesets status
pubm changesets status --verbose
```

`status`는 어떤 패키지가 올라가는지, 각 패키지의 최고 bump 타입이 무엇인지 보여줍니다.

## 버전 적용 미리보기

```bash
pubm changesets version --dry-run
```

다음을 검증할 때 가장 안전합니다.

- 계산된 버전
- 패키지별 changelog 섹션
- fixed / linked 그룹 전파
- prerelease 번호

## changeset 소비

```bash
pubm changesets version
```

이 명령은 changeset을 읽고, 패키지 버전을 계산하고, changelog를 생성하며, 소비한 파일을 지우고, 버전 커밋을 만듭니다.

## changelog만 생성

```bash
pubm changesets changelog --dry-run
pubm changesets changelog --version 1.8.0
```

## `.changeset/`에서 마이그레이션

```bash
pubm changesets migrate
```
