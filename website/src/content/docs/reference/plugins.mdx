The plugin system lets you extend `pubm` without forking the core publish pipeline.

## `PubmPlugin`

```ts
export interface PubmPlugin {
  name: string;
  registries?: Registry[];
  ecosystems?: Ecosystem[];
  hooks?: PluginHooks;
  commands?: PluginCommand[];
}
```

## Hooks

```ts
export interface PluginHooks {
  beforeTest?: HookFn;
  afterTest?: HookFn;
  beforeBuild?: HookFn;
  afterBuild?: HookFn;
  beforeVersion?: HookFn;
  afterVersion?: HookFn;
  beforePublish?: HookFn;
  afterPublish?: HookFn;
  beforePush?: HookFn;
  afterPush?: HookFn;
  afterRelease?: AfterReleaseHookFn;
  onError?: ErrorHookFn;
  onRollback?: HookFn;
  onSuccess?: HookFn;
}
```

## Hook signatures

```ts
type HookFn = (ctx: Ctx) => Promise<void> | void;
type ErrorHookFn = (ctx: Ctx, error: Error) => Promise<void> | void;
type AfterReleaseHookFn = (
  ctx: Ctx,
  releaseCtx: ReleaseContext,
) => Promise<void> | void;
```

## `Ctx`

Hooks receive the current pipeline context. The runtime type is based on resolved CLI options plus a few internal flags.

```ts
export interface Ctx extends ResolvedOptions {
  promptEnabled: boolean;
  cleanWorkingTree: boolean;
  pluginRunner: PluginRunner;
  releaseContext?: ReleaseContext;
}
```

Useful fields from `ResolvedOptions` include:

- `version`
- `versions`
- `packages`
- `tag`
- `preview`
- `preflight`
- `ci`
- `changesetConsumed`

Treat `Ctx` as runtime state for the current pipeline execution, not as a persistence layer.

## Example

```ts
const notifyPlugin = {
  name: "notify-plugin",
  hooks: {
    async afterPublish(ctx) {
      const label =
        ctx.versions && ctx.versions.size > 0
          ? [...ctx.versions].map(([name, version]) => `${name}@${version}`).join(", ")
          : `v${ctx.version}`;

      console.log(`Published ${label}`);
    },
  },
};
```

## Custom commands

```ts
export interface PluginCommand {
  name: string;
  description: string;
  subcommands?: PluginSubcommand[];
}

export interface PluginSubcommand {
  name: string;
  description: string;
  options?: PluginCommandOption[];
  action: (args: Record<string, unknown>) => Promise<void>;
}
```

## Custom registries and ecosystems

Use custom registries when you need another publish destination. Use custom ecosystems when you need to teach `pubm` how to read and write versions for another package type.
