Official plugins are maintained by the `pubm` core team and provide reliable, well-tested integrations with popular tools and workflows.

## `@pubm/plugin-brew`

Automatically updates Homebrew formulas when you release a new version of your CLI tool.

### Features
- **Automatic formula generation**: Generates a valid Ruby formula if one doesn't exist.
- **Resource management**: Detects and updates `url` and `sha256` for release assets (e.g. `darwin-arm64`, `linux-x64`).
- **Git automation**: Commits and pushes the formula to a dedicated tap repository or your local one.
- **PR fallback**: If a branch is protected, it creates a Pull Request instead of pushing directly.

### Configuration

```ts
import { brewTap } from "@pubm/plugin-brew";

export default defineConfig({
  plugins: [
    brewTap({
      // Path to the formula file (relative to repository root)
      formula: "Formula/my-tool.rb",
      // Optional: Remote repository URL for separate Homebrew tap.
      // If omitted, it updates the formula in the current repository.
      repo: "https://github.com/my-org/homebrew-tap",
    }),
  ],
});
```

### Commands
- `pubm brew init`: Scaffolds a new Homebrew formula based on your `package.json` data (name, description, homepage, license).

## `@pubm/plugin-external-version-sync`

Keeps non-standard files in sync with your project's version. Ideal for source-code constants, READMEs, or other manifest types (e.g., `bower.json`, `app.json`).

### Features
- **JSON Path Support**: Precisely update values in any JSON file.
- **Regex Support**: Find and replace versions in any text file using capturing groups.
- **Multi-target**: Sync an unlimited number of files in a single release.

### Configuration

```ts
import { externalVersionSync } from "@pubm/plugin-external-version-sync";

export default defineConfig({
  plugins: [
    externalVersionSync({
      targets: [
        // Update a constant in a TypeScript file
        {
          file: "src/constants.ts",
          type: "regex",
          pattern: /export const VERSION = "(.*)";/,
        },
        // Update version in a JSON file at a specific path
        {
          file: "manifest.json",
          type: "json",
          jsonPath: "$.metadata.version",
        },
        // Update a badge in your README
        {
          file: "README.md",
          type: "regex",
          pattern: /img\.shields\.io\/badge\/version-(.*)-blue/,
        }
      ],
    }),
  ],
});
```

### Options
- `file`: Path to the target file.
- `type`: Either `"regex"` or `"json"`.
- `pattern` (Regex): A regular expression with exactly **one** capturing group for the version string.
- `jsonPath` (JSON): A JSON path (standard notation) to the version property.
